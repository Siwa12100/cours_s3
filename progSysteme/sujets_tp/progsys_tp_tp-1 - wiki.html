Gestion des processus

Exercice n°1

Écrivez un programme qui affiche :

  • tous ses arguments (précédés de leur numéro)
  • une ligne de ----
  • la valeur de la variable d'environnement PATH

Exemple de trace d'exécution : 

# ./ex-1 aaa bbb "ccc ddd"
arg[0]=./ex1
arg[1]=aaa
arg[2]=bbb
arg[3]=ccc ddd
--------------------------------------------------------
PATH => /usr/lib/distcc:/usr/local/bin:/usr/bin:/bin:/usr/games

Remarque : Compilez en demandant au compilateur d'afficher tous les avertissements possibles via une ligne telle que : 

gcc -Wall -pedantic -o ex1 ex1.c

Exercice n°2

Complétez le programme suivant pour qu'il réalise un appel à la commande ps -f (via la fonction system(3)) puis ensuite qu'il effectue un recouvrement par la même commande ps -f (via la fonction execlp(3)). Commentez l'affichage ! 

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
 
int main(void) {
 
        // A compléter
 
        printf("Terminé !\n");
 
        exit(0);
}

Exercice n°3

Soit le squelette de programme suivant : 

#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <unistd.h>
 
 
int main(int argc, char **argv) {
 
       pid_t pid;
 
       switch(pid=fork()) {
 
               case -1 : // Oups !!! fork n'a pas marché !
                         perror("fork"); exit(errno);
 
               case  0 : // Code du fils
                         // A compléter ...
                         break;
 
               default : // Code du père
                         // A compléter...
                         printf("Fin du processus père de pid %d.\n", getpid());
       }
 
 
 
       exit(0);
}

1. Complétez le programme pour que le processus fils créé :

  • affiche les entiers naturels de 0 à 9 avec une temporisation d'une demi-seconde entre chaque (cf. nanosleep(2)),
  • puis qu'il exécute la commande ps -f,
  • et enfin qu'il se termine en retournant 2 comme code de retour.

2. Complétez votre programme pour que cette fois le père attende la fin de son fils (cf. wait(2)) et affiche son code de retour.

Exercice n°4

Cet exercice est une variante du précédent dans lequel le processus père crée N fils qui devront s'exécuter en parallèle. Chaque fils doit afficher son identité (entier élément de [0…n-1]) avant chaque message (affichage des entiers naturels de 0 à 9). Dans un premier temps, N sera une valeur constante, puis vous utiliserez une valeur passée en argument sur la ligne de commande.
